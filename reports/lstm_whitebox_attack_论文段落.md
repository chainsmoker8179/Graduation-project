# 原始 OHLCV 空间下的 LSTM 白盒攻击链路与结果分析

## 方法：攻击链路重建

为在不重新训练旧版基线模型的前提下实施基于梯度的白盒攻击，本文首先重建了一条从原始市场输入到旧版 LSTM 预测器的可微推理链路。旧版基线并不直接消费原始 `OHLCV`，而是以 20 个经 `FilterCol`、`RobustZScoreNorm` 和 `Fillna` 处理后的 Alpha158 特征序列作为输入。因此，攻击若直接作用于原始输入，关键问题不在于攻击算法本身，而在于如何恢复一条语义上尽可能接近旧版 Qlib 流水线、同时又能够保留 Torch 自动求导能力的前向与反向通路。基于这一约束，本文将攻击图拆分为四个顺序模块：原始 `OHLCV` 窗口、20 个旧版特征的可微桥接层、Torch 版稳健归一化与缺失值修复层，以及加载历史 `state_dict` 的旧版 LSTM 预测器。

设原始输入窗口为 \(X \in \mathbb{R}^{80 \times 5}\)，其中五个通道分别对应开盘价、最高价、最低价、收盘价和成交量。之所以将原始窗口长度固定为 80，是因为旧版模型在特征域上使用长度为 20 的时序输入，而所保留的 20 个 Alpha158 特征中存在最长为 60 的滚动依赖；因此，只有在原始输入侧提供 80 个交易日的观测，桥接层才能输出长度为 20 的特征时序张量 \(F(X) \in \mathbb{R}^{20 \times 20}\)。在实现上，桥接层复用了现有的可微 Alpha158 图执行器，并严格保持旧版 20 个特征的顺序不变，从而使后续加载的 LSTM 权重能够直接作用于重建后的特征张量，而无需重新训练或重新排列输入维度。

为了保持与旧版预处理链的一致性，本文进一步采用“Qlib 离线拟合、Torch 在线复现”的方式处理归一化和缺失值修复。具体而言，`RobustZScoreNorm` 的中心和尺度参数由 Qlib 在训练期数据上离线拟合后导出，其中中心对应特征的中位数，尺度对应中位绝对偏差乘以 1.4826，并保留 `clip_outlier=True` 的截断设置；在线攻击阶段则在 Torch 中以冻结参数的方式复现同一归一化公式，并在归一化后执行零值 `Fillna`。这种拆分方式的作用在于，将依赖历史统计量的处理步骤保留在离线阶段完成，同时避免在攻击循环内部调用 Qlib processor，从而保证攻击图始终停留在可微的 Torch autograd 图中。

在预测器侧，本文严格复刻旧版 `qlib.contrib.model.pytorch_lstm_ts.LSTM` 的核心结构，并直接加载迁移得到的 `state_dict`。该预测器的输入维度为 20，隐层维度为 64，堆叠层数为 2，`dropout` 为 0，输出端采用最后一个时间步的隐藏状态经线性层映射得到标量预测。由此，完整的攻击目标可写为在原始输入空间中最大化预测误差：

\[
\max_{\delta} \ \mathcal{L}(X+\delta, y)=\mathrm{MSE}(f(X+\delta), y),
\]

其中 \(f(\cdot)\) 表示“原始 `OHLCV` -> 特征桥接 -> 稳健归一化 -> `Fillna` -> 旧版 LSTM”的组合映射，\(y\) 为与旧实验对齐得到的标签。该设计的核心意义在于，它并未改变被攻击模型本身，而是在尽可能保持旧版语义的前提下，将原本不可直接攻击的因子型输入链重构为可在原始输入空间上实施白盒优化的端到端图。

## 实验设置：攻击协议与评估方法

攻击实验采用旧版 Qlib 运行中导出的 `pred.pkl` 和 `label.pkl` 作为对齐参照，并仅在能够与这两类产物精确匹配的测试样本上进行小规模 smoke 验证。具体地，本文首先在 `(datetime, instrument)` 多重索引上对历史预测与标签进行交集匹配，然后从匹配结果中随机抽取 8 个样本键，并进一步重建对应的原始 `OHLCV` 序列窗口。由于部分键无法在本地恢复出完整的原始窗口，最终得到 7 个可用于攻击的有效样本，每个样本均对应一个形状为 `(80, 5)` 的原始输入张量。第一版实验不接入组合收益或回测评价，而是聚焦于攻击链路是否可用，因此评估对象仅限于模型级别的预测误差与输入扰动行为。

在进入攻击步骤之前，本文引入了一个 clean gate 以验证重建后的推理链是否具备最基本的攻击条件。该 gate 包含四项检查：其一，桥接后的 20 个特征必须全部为有限值；其二，clean 预测与 clean loss 必须可以正常计算；其三，`loss.backward()` 必须能够在原始 `OHLCV` 输入侧产生非空、非 NaN 且平均绝对值大于 0 的梯度；其四，重建后的 clean 预测应与旧版 `pred.pkl` 保持同一数值量级，并在排序层面具有可计算的一致性指标。只有当这四项条件全部满足时，样本才会进入后续 FGSM 与 PGD 攻击流程。该设置的目的不是证明 clean 重建已经完全等价于旧版流水线，而是将“梯度链路断裂”和“语义复现尚不充分”两类失败模式明确区分开来。

在扰动约束上，本文直接在原始 `OHLCV` 空间施加相对 `L_\infty` 约束。对于任意时间步 \(t\) 和通道 \(c\)，扰动上界被定义为当前值幅度的固定比例，即 \(|\delta_{t,c}| \le \epsilon_c \cdot \max(|X_{t,c}|, b_c)\)。其中，四个价格通道共用同一预算，实验中设置为 `price_epsilon=0.01`；成交量通道单独设置预算，实验中设置为 `volume_epsilon=0.02`。每一步更新后，扰动均需要重新投影回以 clean 输入为中心的相对约束盒，并同时满足非负域约束。第一版实验不额外施加 `high \ge \max(open, close)` 或 `low \le \min(open, close)` 等 K 线结构一致性约束。FGSM 采用单步符号梯度更新；PGD 采用 5 步迭代更新，单步步长设为预算的 0.25，且不使用随机初始化。评估指标包括 `clean_loss`、`adv_loss`、预测偏移 \(|\hat{y}_{adv}-\hat{y}_{clean}|\)、实际预算使用比例，以及以 `adv_loss > clean_loss` 定义的轻量攻击成功指示。

## 结果：攻击链路可用性与小样本攻击效应

小样本结果首先验证了攻击链路在原始输入空间中的可微性。对于最终保留的 7 个有效样本，clean gate 全部通过：特征有限值比例为 1.0，输入梯度有限值比例也为 1.0；输入梯度的平均绝对值为 \(3.05 \times 10^{-5}\)，最大绝对值为 \(4.01 \times 10^{-3}\)。这些结果表明，经过“原始 `OHLCV` -> 20 特征桥接 -> 稳健归一化 -> 旧版 LSTM”的重建后，监督损失已经能够稳定地回传到原始输入端。换言之，本阶段的主要技术目标，即在不重训模型的前提下打通原始输入上的梯度通路，已经在真实样本上得到直接验证。

在 clean 预测与旧版参考预测的一致性方面，当前重建结果仍然表现为“链路成立但复现尚未完全收敛”的状态。具体而言，7 个样本上的 clean loss 为 0.003037，clean 预测均值与标准差分别为 -0.0193 和 0.0510，而旧版参考分数的均值与标准差分别为 0.0512 和 0.0768；二者的 Spearman 排序相关系数为 0.4286。该结果说明，当前 Torch 重建链已经能够在量级上产生合理输出，并保留一定的排序一致性，但与旧版 Qlib 运行之间仍存在可见偏差。因此，后续若要将攻击结论进一步外推到完整测试集或投资组合层面，仍需要继续压缩离线与在线流水线之间的语义差异。

尽管 clean 对齐仍有提升空间，FGSM 和 PGD 在当前小样本上已经能够稳定提升预测误差。以 clean 推理为参照，FGSM 将均方误差从 0.003037 提高到 0.032132，PGD 则进一步提高到 0.056649；对应的平均绝对预测偏移分别为 0.1249 和 0.1936。在以 `adv_loss > clean_loss` 作为攻击成功判据的设置下，FGSM 与 PGD 在该 7 个样本组成的 smoke 子集上均满足成功条件。进一步观察样本级结果可以发现，FGSM 在 7 个样本上的绝对预测偏移范围为 0.0452 到 0.1717，PGD 的绝对预测偏移范围为 0.1195 到 0.2515。这说明当前攻击并非仅在个别样本上产生孤立的大幅扰动，而是在整个有效样本子集中均能够诱发明显的预测偏移。

从预算使用情况看，当前攻击在成交量维度上表现出更高的边界利用率。FGSM 的价格通道平均预算使用率为 0.4286，而成交量通道平均使用率达到 1.0；PGD 的价格通道平均使用率为 0.3228，成交量通道平均使用率为 0.7844，二者的最大使用率都接近 1.0。该结果表明，在当前重建链路和小样本设置下，基于梯度的扰动更新倾向于更充分地利用成交量维度的可行域，而价格维度上的有效更新则更集中于部分时间步和通道。结合前述攻击成功结果，可以初步认为，至少在当前 smoke 场景中，成交量相关梯度已经能够被攻击器有效利用，但这一现象是否具有更广泛的稳定性，仍需要在更大样本和更严格 clean 对齐条件下进一步验证。

## 讨论：当前结论与边界

综合以上结果，本文当前更合适的结论不是“原始 `OHLCV` 白盒攻击已经被完全解决”，而是“在保留旧版模型参数的条件下，原始输入上的攻击通路已经被成功构建，并在小样本上显示出可观测的误差放大效应”。支撑这一判断的证据主要来自两个方面：一是 clean gate 中输入梯度非空、有限且均值大于 0，说明梯度链路本身已经成立；二是相对于 clean 推理，FGSM 与 PGD 均能在相同约束下将损失提升一个数量级以上，说明该梯度链路已经足以支持实际的白盒优化。然而，当前实验仍受三个边界条件限制。首先，样本规模仅为 7 个可重建窗口，结果更适合被理解为功能性验证而非统计性结论。其次，clean 预测与旧版参考预测的 Spearman 仅为 0.4286，说明离线 Qlib 流水线与在线 Torch 复现之间仍存在残余偏差。最后，本轮实验尚未接入投资组合收益、风险暴露和回测指标，因此现阶段还不能直接回答攻击对下游交易决策的影响。

基于上述边界，后续工作应沿两个方向继续推进。第一，需要进一步提升 clean 重建的保真度，例如在特征级别逐项对齐旧版流水线输出，并继续排查样本窗口、缺失值处理与统计量应用中的细粒度偏差。第二，在 clean 对齐改善后，应将当前 smoke 攻击扩展到更大的测试子集，并进一步评估攻击后的信号排序变化、组合收益退化和风险指标变化。只有在完成这两个步骤之后，才能将“梯度可达”稳健地转化为“对量化决策有效的白盒对抗攻击”。

## 扩展实验：部分股票攻击的组合级比例敏感性

在完成攻击链路修复与样本级攻击验证之后，本文进一步考察局部白盒扰动是否会传导到组合构建层面，以及这种退化是否会随着攻击覆盖率上升而增强。为此，本文在 legacy LSTM 基线模型上开展部分股票攻击比例对比实验，测试区间覆盖 `2025-01-01` 至 `2025-10-31`，并在每个交易日内随机选择固定比例的股票施加扰动，其中攻击比例设置为 `1%`、`5%` 和 `10%`，随机种子取 `0` 至 `4`。攻击目标统一定义为最大化均方误差，攻击器采用 FGSM 与 PGD。为避免不同攻击比例下可攻击样本覆盖差异污染结论，本文首先对三档比例和五个随机种子的候选攻击键取并集，并据此导出一份共享的 raw-only 资产。该共享资产包含 `591,791` 个候选键，其中 `590,504` 条样本可与参考预测对齐，最终导出 `574,386` 条可用原始窗口；在这一统一资产口径下，三档比例实验分别获得平均 `10,281.0`、`50,875.0` 和 `101,670.8` 条可攻击样本，且标准差仅为 `17.35`、`18.03` 和 `34.65`，说明多随机种子下的样本覆盖波动较小。

组合级评价不直接以原始 `reference_clean` 为主对照，而是采用 `partial_clean` 隔离“局部替换 clean 分数”本身带来的拼接偏差，并分别比较 `partial_fgsm - partial_clean` 与 `partial_pgd - partial_clean`。结果显示，即使仅对局部股票子集施加对抗扰动，组合级回测指标仍会出现稳定退化，并且退化幅度随攻击比例上升而系统增强。对于 FGSM，相对 `partial_clean` 的年化超额收益率退化均值从 `1%` 比例下的 `-0.2026` 扩大到 `5%` 下的 `-0.2531`，并进一步扩大到 `10%` 下的 `-0.3313`；对应的信息比率退化从 `-1.3538` 扩大到 `-1.7747` 和 `-2.3932`，`RankIC` 均值退化则从 `-0.002761` 持续扩大到 `-0.013098` 和 `-0.025438`。PGD 呈现出相同但更强的趋势：其年化超额收益率退化均值分别为 `-0.3014`、`-0.3624` 和 `-0.4128`，信息比率退化均值分别为 `-2.1382`、`-2.6983` 和 `-3.0935`，`RankIC` 均值退化分别为 `-0.003078`、`-0.014508` 和 `-0.028142`。在最大回撤指标上，FGSM 的平均退化为 `-0.0426`、`-0.0493` 和 `-0.0500`，PGD 的平均退化为 `-0.0438`、`-0.0544` 和 `-0.0661`，也整体表现出更高攻击比例对应更差风险暴露的趋势。

从跨种子稳定性看，FGSM 和 PGD 的四项主指标在三档比例、五个随机种子上的方向均高度一致。对于 FGSM，相对 `partial_clean` 的四项核心差值在每个比例下均为 `5/5` 个种子为负；对于 PGD，同样四项指标在每个比例下也均为 `5/5` 个种子为负。这说明局部攻击导致的组合退化并非来自个别随机抽样，而是在共享资产口径下具有稳定复现性。进一步比较两类攻击器可以发现，PGD 在收益类与排序类指标上持续强于 FGSM。具体而言，在全部 `15` 个 `ratio-seed` 组合中，PGD 对年化超额收益率、`RankIC` 均值和信息比率的退化均始终大于 FGSM；在最大回撤上，PGD 在 `1%` 和 `5%` 比例下分别在 `3/5` 个种子上更差，在 `10%` 比例下则在 `5/5` 个种子上更差。对 FGSM 与 PGD 分别考察年化超额收益率、最大回撤、`RankIC` 均值和信息比率四项指标，可以发现全部 `8` 组比例趋势检查都满足 `1% >= 5% >= 10%` 的均值次序，其中“更小”表示相对 `partial_clean` 的退化更强。该结果说明，对抗扰动在截面上的覆盖率会直接影响其向组合构建层面的传导强度，攻击比例越高，更多股票的排序信号被系统性偏移，组合最终承受的收益下降和风险恶化也越明显。

需要指出的是，本轮比例对比并未被 clean 拼接误差主导。三档比例下，`partial_clean - reference_clean` 的残余偏差整体明显小于攻击差值。例如在 `10%` 比例下，`partial_clean - reference_clean` 的年化超额收益率均值仅为 `-0.0041`，而 `partial_fgsm - partial_clean` 与 `partial_pgd - partial_clean` 的对应均值分别达到 `-0.3313` 和 `-0.4128`。类似地，`RankIC` 与信息比率上的 clean 残余项也均远小于对抗退化项。因此，本文认为当前观察到的比例趋势主要来自攻击覆盖率提高后扰动对组合排序的累积影响，而不是来自局部 clean 替换本身。综合样本级攻击验证与组合级比例对比结果，可以得到一个更完整的结论：原始 `OHLCV` 空间中的白盒扰动不仅能够放大模型预测误差，而且能够在仅攻击部分股票的条件下传导到投资组合层面，并随着攻击覆盖率上升而系统性削弱组合收益质量。
