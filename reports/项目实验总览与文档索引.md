本文档用于把当前 worktree `.worktrees/multi-model-whitebox-attack` 中已经形成的实验主线、阶段结论和 Markdown 文档做一次统一整理，主要回答四个问题：

3. 截至目前，哪些结果应视为当前有效基线，哪些只是过程性版本或排查性产物。
4. 如果现在重新接手这个项目，建议先读哪些文档，才能最快进入状态。

## 3. 实验主线的逻辑先后关系

如果把当前项目压缩成一条主线，可以概括为：

`可导化设计 -> BPDA 算子级验证 -> 端到端梯度验证 -> LSTM 原始 OHLCV 白盒攻击设计 -> LSTM 小样本 smoke -> LSTM 扩样与 clean gate 收紧 -> clean 偏弱根因定位 -> LSTM 正式样本级基线 v6 -> partial attack 回测 -> 多随机种子稳定性与比例趋势 -> LSTM 统计显著性与排序机制补强 -> Transformer/TCN clean rebuild probe -> Transformer/TCN 小样本迁移 -> Transformer/TCN 4096 样本正式实验`

这条主线里其实有两条分支在并行推进：

1. `LSTM 组合级分支`
从 `expanded_v6` 样本级结果继续往下，验证局部股票攻击是否会传导到组合收益、回撤和 RankIC 退化，并进一步补齐显著性检验和排序机制分析。

2. `多模型迁移分支`
在 LSTM 样本级链路跑通并稳定之后，把同一套 white-box attack 迁移到 Transformer 和 TCN，上探“攻击是否具有跨模型一致性”。

因此，当前项目并不是单线程推进，而是在“LSTM 做深”和“多模型做广”两条线上同步积累证据。

## 4. 各阶段实验的作用、问题与衔接关系

### 4.1 阶段 A：可导特征模块建设

对应文档：

- `可导特征模块章节.md`
- `plan.md`

这一阶段回答的是方法层问题：为什么传统 Alpha158 会阻断梯度，如果希望以后做端到端训练或梯度型攻击，应如何把这些因子表达式改造成可导图。

这一阶段的产出不是具体实验结论，而是后续所有实验的共同底座。没有这一步，后续无论是端到端梯度验证还是 FGSM/PGD 攻击，都没有可靠的可微因子图可用。

这个阶段留下的核心待验证问题是：设计上可导，不代表局部算子级别真的可信，因此自然进入下一阶段 BPDA 验证。

### 4.2 阶段 B：BPDA 算子级梯度验证

对应文档：

- `scripts/BPDA_GRAD_VALIDATION.md`
- `reports/validate_bpda_gradients_中文详解报告.md`

这一阶段回答的问题是：对 `max/min`、比较、分位数、`idxmax/idxmin`、`rank` 等离散或非光滑算子，BPDA 是否同时满足“前向硬语义一致”和“反向软近似可提供合理梯度”。

这一阶段的主要结论是：

1. BPDA 前向与硬算子保持一致。
2. BPDA 反向梯度与软替代算子的梯度趋势一致。
3. 梯度方向性、边界敏感性和单调性基本符合预期。

这一步建立的是局部可信性，但仍不能推出真实市场数据上的整条链路一定可回传梯度，因此后续进入端到端验证。

### 4.3 阶段 C：端到端梯度回传验证

对应文档：

- `reports/e2e_grad_smoke/e2e_grad_validation_report.md`
- `reports/e2e_grad/e2e_grad_validation_report.md`
- `reports/e2e_grad_validation.md`

这一阶段回答的问题是：在真实 Qlib 数据上，监督损失是否真的能够跨过完整的可微因子提取器，稳定回传到原始 OHLCV 输入。

主要结论是：

1. 完整因子集和离散算子相关因子子集的 batch 通过率都达到 `1.0`。
2. 整条 `OHLCV -> 因子 -> loss` 梯度链路已经被真实数据验证为可用。
3. 但 `volume-diff` 相关的 `VSUMP5 / VSUMN5 / VSUMD5` 表现出局部弱梯度，这为后续攻击实验留下了一个需要关注的风险点。

这个阶段给后续攻击提供了前提：梯度通路已经存在，因此可以尝试把它真正接到历史训练好的旧模型上。

### 4.4 阶段 D：LSTM 原始 OHLCV 白盒攻击链路设计

对应文档：

- `docs/superpowers/specs/2026-03-11-raw-ohlcv-lstm-whitebox-attack-design.md`
- `docs/superpowers/plans/2026-03-11-raw-ohlcv-lstm-whitebox-attack.md`

这一阶段回答的问题是：如何把“可导因子能力”嫁接到历史训练好的 Qlib LSTM 基线，从而在原始 OHLCV 空间里实施 FGSM/PGD，而不是重新训练一个新模型。

难点不在攻击算法本身，而在旧链路重建。旧版 LSTM 消费的是经过 `FilterCol + RobustZScoreNorm + Fillna` 处理后的 20 维特征，而不是原始 OHLCV。

因此，这一阶段真正搭建的是：

`原始 OHLCV -> 20 个旧版特征 -> Torch 版 RobustZScoreNorm/Fillna -> 旧版 LSTM`

到这一步为止，设计已经完整，但还缺少实验层证据，所以进入小样本 smoke。

### 4.5 阶段 E：LSTM 小样本 smoke 攻击

对应文档：

- `reports/lstm_whitebox_attack_smoke_v1/attack_report.md`

这一阶段回答的问题是：在极小规模、便于人工核对的样本子集上，这条攻击链路是否真的能跑通，FGSM/PGD 是否至少能抬高损失。

主要结论是：

1. clean gate 可以通过，说明梯度确实能回传到原始 OHLCV。
2. FGSM 和 PGD 都能显著提高 MSE，说明攻击方向是有效的。
3. 但 clean 对齐偏弱，排序对齐度还不够强，尚不足以支撑更大规模正式结论。

因此，项目下一步不是直接写论文，而是先扩大样本并收紧 clean gate。

### 4.6 阶段 F：LSTM 扩样与 clean gate 收紧

对应文档：

- `docs/superpowers/plans/2026-03-11-attack-clean-gate-and-expanded-smoke.md`
- `reports/lstm_whitebox_attack_expanded_probe_v1/attack_report.md`
- `reports/lstm_whitebox_attack_expanded_v2/attack_report.md`
- `reports/lstm_whitebox_attack_expanded_v3/attack_report.md`

这一阶段回答的问题是：当样本规模扩大，且 clean gate 不只检查梯度存在，还加入中间特征对齐指标后，这条攻击链路是否仍然成立。

这一阶段的关键价值不是“结果多好”，而是暴露了一个更深的问题：

1. 中间特征的 MAE/RMSE 看起来并不大。
2. 但最终预测与 `pred.pkl` 的排序对齐却明显偏弱。

这说明问题不像“公式全错了”，更像是存在某种更隐蔽的语义错位，因此项目转入根因定位阶段。

### 4.7 阶段 G：LSTM clean 偏弱根因定位

对应文档：

- `reports/lstm_whitebox_attack_expanded_v5_quick/attack_report.md`
- `reports/lstm_whitebox_attack_expanded_report.md`

这一阶段回答的问题是：为什么特征误差不大，但最终模型输出排序对齐很差。

最终定位出的主因是：20 维特征顺序错位。

具体来说：

1. 训练时 Qlib `FilterCol` 只做筛列，不会按手写 `feature_cols` 重新排序。
2. 攻击链路此前误把手写顺序当成真实输入顺序。
3. 结果就是“数值看起来像，但维度语义错位”，最终造成 clean 偏弱。

这个阶段非常关键，因为它说明问题不是模型权重、不是预测文件，也不是特征公式整体错误，而是输入维度语义错位。

### 4.8 阶段 H：LSTM 正式样本级基线形成

对应文档：

- `reports/lstm_whitebox_attack_expanded_v6/attack_report.md`
- `reports/lstm_whitebox_attack_expanded_report.md`
- `reports/lstm_whitebox_attack_论文段落.md`

这一阶段回答的问题是：在修复特征顺序错位之后，clean 对齐是否显著提升，FGSM/PGD 的攻击效应是否同步增强。

主要结论是：

1. `expanded_v6` 的 clean 对齐显著好于 `v2/v3`。
2. 在 clean 对齐提升后，FGSM 和 PGD 对预测误差的放大也更强。
3. 因此 `expanded_v6` 应被视为当前 LSTM 样本级攻击的正式基线。

到这一步，项目已经不再只是“链路存在性验证”，而是形成了可引用的正式样本级攻击结果。

### 4.9 阶段 I：partial attack 回测从失败 smoke 到正式基线

对应文档：

- `reports/partial_attack_backtest_smoke_v6/README.md`
- `reports/partial_attack_backtest_seed0_ratio5_v1/README.md`
- `reports/partial_attack_backtest_seed0_ratio5_v1_实验报告.md`

这一阶段回答的问题是：即使不攻击全市场，只在每个交易日随机攻击一部分股票，攻击扰动是否仍会传导到组合级收益、风险和排序指标。

这个阶段分成两个子阶段：

1. `partial_attack_backtest_smoke_v6`
早期方案几乎无法对齐攻击样本，`attackable_count=0`，因此没有形成有效攻击。这一步很重要，因为它说明原先拼接方式不可用，逼着后续重新设计共享资产与对齐流程。

2. `seed0_ratio5_v1`
重新设计后，单种子 `5%` 实验成功证明：即使只攻击每日局部股票，组合年化超额收益会下降，最大回撤会恶化，RankIC 也会下降。

这一步把“样本级误差放大”推进成了“组合级表现退化”。

### 4.10 阶段 J：partial attack 的稳定性与比例趋势验证

对应文档：

- `docs/superpowers/specs/2026-03-13-partial-attack-ratio-sweep-design.md`
- `docs/superpowers/plans/2026-03-13-partial-attack-ratio-sweep.md`
- `reports/partial_attack_backtest_multiseed_ratio5_union/多随机种子稳定性报告.md`
- `reports/partial_attack_ratio_sweep_multiseed/攻击比例对比报告.md`
- `reports/partial_attack_ratio_sweep_multiseed/攻击比例对比_论文段落.md`

这一阶段回答的问题是：单种子 `5%` 下观察到的组合级退化，究竟是偶然现象，还是在更多随机种子和更多攻击比例下都能稳定复现。

主要结论是：

1. `5%` 比例下，多随机种子复跑显示退化方向稳定，不是单次抽样偶然。
2. 当攻击比例从 `1%` 提高到 `5%` 再到 `10%` 时，收益、排序和风险指标的退化整体呈增强趋势。
3. PGD 通常比 FGSM 更强，这一结论在组合层面仍成立。

至此，LSTM 攻击已经从存在性证明升级为具有稳定性和趋势性的金融实验结果。

### 4.11 阶段 K：LSTM 单模型证据补强

对应文档：

- `docs/superpowers/specs/2026-03-17-lstm-whitebox-evidence-hardening-design.md`
- `docs/superpowers/plans/2026-03-17-lstm-whitebox-evidence-hardening.md`
- `reports/lstm_single_model_evidence_report.md`

这一阶段回答的问题是：已有 LSTM 结果是否已经足够支撑论文与答辩，是否能从“均值退化”进一步提升到“统计显著且机制清晰”。

这一阶段做的是证据强化，而不是新攻击链路设计。其作用主要有三点：

1. 补上日度超额收益与 RankIC 的显著性检验。
2. 增加 Top-K 重合率、分数 Spearman、排名位移等排序机制分析。
3. 给出主文与附录的结果组织建议。

因此，`lstm_single_model_evidence_report.md` 应被视为当前 LSTM 论文证据的高层摘要。

### 4.12 阶段 L：Transformer / TCN clean rebuild probe

对应文档：

- `reports/transformer_model_probe/README.md`
- `reports/tcn_model_probe/README.md`

这一阶段回答的问题是：在把攻击迁移到其他模型之前，本地 Torch 复现链路能否在 clean forward 上精确重建旧的 Transformer / TCN 预测输出。

主要结论是：

1. 两个模型的 `mae_to_reference` 与 `mse_to_reference` 都极小。
2. `spearman_to_reference = 1.0`。

这一步的作用类似 LSTM 的“clean 对齐前置证明”，只是这里直接证明迁移的基础条件已经满足。

### 4.13 阶段 M：Transformer / TCN 样本级攻击迁移

对应文档：

- `reports/transformer_whitebox_attack_smoke/attack_report.md`
- `reports/transformer_whitebox_attack_expanded_v1/attack_report.md`
- `reports/transformer_whitebox_attack_all_available_v1/attack_report.md`
- `reports/tcn_whitebox_attack_smoke/attack_report.md`
- `reports/tcn_whitebox_attack_expanded_v1/attack_report.md`
- `reports/tcn_whitebox_attack_all_available_v1/attack_report.md`

这一阶段回答的问题是：统一的 `run_whitebox_attack.py` 是否真的能迁移到 Transformer 与 TCN，并在小样本到中等样本范围内稳定抬高预测误差。

主要结论是：

1. 两个模型都已通过 smoke、62 样本、124 样本三个层次的样本级验证。
2. clean gate 在两个模型上均稳定通过。
3. FGSM 和 PGD 在两个模型上都能稳定提高 MSE，且 PGD 更强。

这一步说明攻击不是只对 LSTM 有效，已经具备跨模型迁移能力。

### 4.14 阶段 N：Transformer / TCN 4096 样本正式实验与快导出固化

对应文档：

- `docs/superpowers/specs/2026-03-20-multi-model-large-sample-whitebox-eval-design.md`
- `docs/superpowers/plans/2026-03-20-multi-model-large-sample-whitebox-eval.md`
- `docs/superpowers/specs/2026-03-20-fast-large-sample-attack-export-design.md`

- `reports/tcn_whitebox_attack_4096_v1_fast/attack_report.md`

- `reports/tcn_whitebox_attack_4096_v1/attack_report.md`
- `reports/transformer_tcn_4096白盒攻击实验报告.md`

这一阶段回答的问题有两个：

1. Transformer / TCN 的攻击结论在约 `4k` 样本规模上是否仍然稳定成立。
2. 旧的 Qlib 慢导出路径过慢时，是否可以把“从 raw window 直接重建 feature window”的快导出流程正式脚本化，而不改变攻击定义。

1. 两个模型在 `3983` 个有效样本规模上都通过了 clean gate。

这一步的意义在于：多模型迁移不再停留在 `32/62/124` 这类小样本验证，而是已经形成正式大样本样本级基线。

## 7. 截至目前的项目状态总结

截至目前，可以把项目进展概括为以下几点：

1. 可导 Alpha158 的方法底座已经建立，BPDA 和相关平滑近似不再只是概念，而是已经进入可执行、可验证的工程方案。
2. BPDA 在离散算子级别和真实数据端到端级别都已经完成验证，说明梯度链路是存在且可用的。
3. 面向历史已训练 LSTM 基线的原始 OHLCV 白盒攻击链路已经稳定跑通，并经历了“扩样暴露问题 -> 定位特征顺序错位 -> 修复后形成正式基线”的完整闭环。
4. 在 LSTM 上，攻击证据已经从样本级误差放大，推进到了组合级收益/风险/排序退化，并进一步补上了多随机种子、比例趋势和显著性/排序机制分析。
5. 在多模型方向，Transformer 与 TCN 已经完成 clean rebuild probe、小样本迁移和 `4096` 样本正式实验，说明样本级 white-box attack 已具备跨模型扩展能力。

当前还保留三个自然的后续方向：

1. 把组合级 partial attack backtest 迁移到 Transformer 与 TCN，形成跨模型的组合层证据。
2. 在当前 LSTM 结果基础上继续做更完整的统计检验、收益归因和答辩图表整合。
3. 在样本级攻击中进一步引入金融物理约束或统计约束，研究攻击强度与可检测性的权衡。
