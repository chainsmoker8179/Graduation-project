# 部分股票攻击比例对比设计

## 1. 目标

在已经完成 `5%` 攻击比例、多随机种子稳定性验证之后，进一步比较不同攻击覆盖率对组合级退化的影响。第一版固定：

- 模型：legacy LSTM
- 测试区间：`2025-01-01` 至 `2025-10-31`
- 随机种子：`seed=0..4`
- 攻击比例：`1% / 5% / 10%`
- 攻击目标：最大化 MSE
- 攻击器：FGSM 与 PGD

核心问题是：当每日被攻击的股票比例从 `1%` 提升到 `5%` 再提升到 `10%` 时，组合层面的超额收益、最大回撤、`RankIC` 和信息比率是否出现单调或近似单调退化。

## 2. 设计原则

本轮实验继续以 `partial_clean` 作为主对照，比较：

- `partial_fgsm - partial_clean`
- `partial_pgd - partial_clean`

而不是直接与 `reference_clean` 比较。原因不变：`partial_clean` 可以隔离局部替换 clean 分数本身带来的残余偏差，使不同攻击比例下的退化量更可解释。

## 3. 共享资产策略

最初的 `5%` 多 seed 实验已经暴露出一个关键问题：不能直接复用某个固定 `seed` 或固定 `ratio` 的 partial asset 去跑其他配置，因为资产覆盖范围本身会污染 `attackable_count`。因此，比例对比必须使用共享资产策略。

本轮共享资产按以下方式构造：

1. 在完整测试窗口内，对 `ratio=1% / 5% / 10%` 和 `seed=0..4` 分别生成候选攻击 key。
2. 对全部候选 key 取并集。
3. 仅为这个总并集导出一份共享 raw-only asset，至少包含：
   - `matched_reference.csv`
   - `normalization_stats.json`
   - `matched_ohlcv_windows.pt`

这样可以确保三档比例、五个种子都在同一资产覆盖集合上运行，避免“比例不同只是因为样本能否被攻击不同”这一偏差。

## 4. 实验产出

本轮实验需要产生三类结果：

1. 每个 `ratio-seed` 组合的原始输出目录。
2. 跨 `seed` 的比例汇总表，包括均值、标准差和方向一致性。
3. 中文实验报告，总结攻击比例上升时退化是否增强，以及这种趋势在 FGSM 和 PGD 下是否一致。

## 5. 判定口径

本轮不做统计显著性检验，只做趋势稳定性判断。重点看三类现象：

1. 对固定攻击器，`10%` 的退化是否通常强于 `5%`，`5%` 是否通常强于 `1%`。
2. 同一比例下，PGD 是否仍普遍强于 FGSM。
3. `partial_clean - reference_clean` 是否仍维持在可控范围，避免比例对比被 clean 残差主导。

## 6. 非目标

本轮不做以下扩展：

1. 不新增攻击器。
2. 不改预算、步数、步长。
3. 不新增 feature-level clean gate。
4. 不引入新的模型。
5. 不接入回测之外的行业暴露或换手详细分解。
