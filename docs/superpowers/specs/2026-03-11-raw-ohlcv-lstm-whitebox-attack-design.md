# 原始 OHLCV 输入下的 LSTM 白盒攻击设计

## 背景

当前项目已经具备可微 Alpha158 实现、梯度验证脚本，以及从历史 Qlib 实验迁移出的基线 LSTM 模型产物。这个基线 LSTM 并不是一个直接消费原始输入的端到端模型。它的训练输入是经过 `FilterCol`、`RobustZScoreNorm` 和 `Fillna` 处理后的 20 个 Alpha158 特征，标签则由原有 Qlib 数据流水线生成。当前阶段的目标不是重新训练这一基线模型，而是重建一条可微的 `原始 OHLCV -> 特征 -> 归一化 -> LSTM -> loss` 前向与反向路径，使得基于梯度的白盒攻击可以直接作用于原始市场输入。

用户希望第一轮攻击只聚焦于一小部分可精确对齐的测试样本，在原始 `OHLCV` 空间中使用白盒 `FGSM` 与 `PGD`，在相对扰动约束下最大化预测误差。组合收益评估和回测影响在这一版中明确不纳入范围。

## 目标

构建一个可复现的攻击流程，复用已经迁移出来的基线 LSTM 权重，并支持从原始 `OHLCV` 输入出发的白盒攻击。该流程需要足够准确地重建旧版 20 特征输入链，使得 clean 样本预测与旧基线输出处于同一数值量级，并与导出的 `pred.pkl` 保持较高排序相关性。

## 范围内内容

- 根据导出的配置与 `state_dict` 重建旧版 LSTM 结构。
- 利用仓库中已有的可微 Alpha158 算子，从原始 `OHLCV` 重建旧版使用的 20 个输入特征。
- 从 Qlib 导出并复用旧版 `RobustZScoreNorm` 的拟合统计量。
- 实现一个可放入攻击图中的 Torch 版旧归一化层。
- 将一小部分测试样本与导出的旧版 `pred.pkl` 和 `label.pkl` 对齐。
- 在该匹配子集上运行白盒 `FGSM` 与 `PGD`。
- 输出 clean 与 adversarial 的 loss、预测偏移、扰动使用情况以及轻量级攻击成功率。

## 范围外内容

- 在 clean 重建明显失败前，不重新训练或微调基线 LSTM。
- 不做组合分析、信号回测或下游交易评估。
- 第一版不强制满足 K 线一致性约束，例如 `high >= max(open, close)`。
- 在小样本 smoke 流程稳定前，不做全测试集规模的攻击扫描。

## 旧版基线约束

旧版 Qlib notebook 中的基线 LSTM 定义如下：

- 模型：`qlib.contrib.model.pytorch_lstm_ts.LSTM`
- 特征维度：`d_feat = 20`
- 隐层大小：`64`
- 层数：`2`
- dropout：`0`
- 损失函数：`mse`
- 序列长度：`20`

旧版筛选出的 20 个特征为：

- `RESI5`
- `WVMA5`
- `RSQR5`
- `KLEN`
- `RSQR10`
- `CORR5`
- `CORD5`
- `CORR10`
- `ROC60`
- `RESI10`
- `VSTD5`
- `RSQR60`
- `CORR60`
- `WVMA60`
- `STD5`
- `RSQR20`
- `CORD60`
- `CORD10`
- `CORR20`
- `KLOW`

旧版特征预处理顺序为：

1. `FilterCol`，保留上述 20 个特征
2. `RobustZScoreNorm(clip_outlier=True)`
3. `Fillna`

标签定义保持为：

- `Ref($close, -2) / Ref($close, -1) - 1`

## 拟采用的整体架构

实现方案分为两部分：离线 Qlib 资产导出路径，以及在线 Torch 攻击路径。

### 离线 Qlib 资产导出

Qlib 继续负责那些依赖原始拟合数据状态的操作：

- 加载旧实验引用
- 导出匹配测试样本标识
- 导出 `pred.pkl` 与 `label.pkl` 参照
- 导出 20 个特征对应的 `RobustZScoreNorm` 拟合统计量
- 导出或重建与这些标识对齐的原始 `OHLCV` 测试样本

这一阶段不属于攻击计算图的一部分，只产出后续 Torch 代码需要使用的静态资产。

### 在线 Torch 攻击图

在线推理与攻击图必须对原始 `OHLCV` 保持可微。

前向路径为：

1. 原始 `OHLCV` 样本张量
2. 基于现有 Alpha158 图执行器构建的可微 20 特征提取器
3. 使用旧拟合统计量冻结参数的 Torch `RobustZScoreNorm` 层
4. 与 Torch 执行兼容的缺失值修复
5. 根据导出 `state_dict` 重建出的旧版 LSTM 预测器
6. 标量预测值
7. 与对齐标签计算得到的 `MSE` loss

最关键的边界是：攻击循环内部不允许再调用 Qlib processor。所有攻击时计算必须全部停留在 Torch autograd 图中。

## 数据对齐

只有同时满足以下条件的样本，才允许进入第一版 smoke test：

- 存在于导出的旧版 `pred.pkl` 中
- 存在于导出的旧版 `label.pkl` 中
- 能够在本地重建出语义一致的原始 `OHLCV` 序列窗口

规范主键预期为 Qlib 信号输出常用的 `(datetime, instrument)` 索引。攻击脚本应当先基于这一键构建匹配表，再从中随机抽取一个小子集用于 smoke test。

## Clean 验证闸门

在进入 FGSM 或 PGD 之前，攻击流程必须在采样得到的 clean 子集上通过下面 4 项检查：

1. 重建得到的 20 个特征全部为有限值。
2. clean 预测值与 clean loss 全部可正常计算且为有限值。
3. 重建后的 clean 预测与旧版 `pred.pkl` 处于同一量级，并具有足够高的排序一致性。
4. `loss.backward()` 能产生原始 `OHLCV` 的梯度，且该梯度非空、有限、平均绝对值大于零。

设置这一闸门是为了隔离 3 类失败模式：

- 样本对齐错误
- 特征或归一化重建错误
- 端到端梯度链路断裂

在 clean 闸门未通过前，不应运行攻击代码。

## 攻击定义

### 攻击目标

第一版采用非定向白盒攻击，目标是最大化预测误差：

\[
L(x, y) = \mathrm{MSE}(f(x), y)
\]

其中 `x` 为原始 `OHLCV`，`f` 为重建后的可微流水线，`y` 为对齐得到的旧版标签。

### 扰动空间

扰动直接施加在原始 `OHLCV` 空间。

每个元素允许的扰动上界定义为相对 `L_inf` 约束盒：

\[
|\delta_{t,c}| \le \epsilon_c \cdot \max(|x_{t,c}|, floor_c)
\]

预算分成两个分组：

- `open`、`high`、`low`、`close` 共用一个预算
- `volume` 使用单独预算

### 投影规则

每一步攻击更新后都必须投影回：

1. 围绕 clean 输入定义的相对扰动盒
2. 五个原始输入列的非负域

第一版不强制满足 K 线高低开收关系。

### FGSM

FGSM 使用一次带符号梯度更新，并在更新后立即做投影。

### PGD

PGD 使用多步带符号梯度更新，第一版不采用随机初始化，并在每一步更新后执行投影。

## 评估指标

第一轮攻击仅记录模型级别的攻击结果：

- `clean_loss`
- `fgsm_loss`
- `pgd_loss`
- 预测偏移绝对值 `|y_adv - y_clean|`
- 价格列的实际相对扰动使用情况
- `volume` 列的实际相对扰动使用情况
- 轻量攻击成功指示，定义为 `adv_loss > clean_loss`

第一版不纳入任何组合收益或回测指标。

## 交付物

预期实现产物包括：

- 旧版 LSTM 重建模块
- 20 特征可微提取模块
- Torch 版 `RobustZScoreNorm` 层
- 用于导出归一化统计量和匹配样本参照的 Qlib 资产导出脚本
- 支持 clean、FGSM、PGD 评估的小样本攻击运行脚本
- smoke test 结果的 markdown 或 JSON 汇总输出

## 风险与缓解

### 风险 1：clean 预测无法与旧基线对齐

可能原因：

- 特征不一致
- 归一化不一致
- 样本对齐错误

缓解方式：

- 将中间特征张量与离线导出的参照进行比较
- 校验 `RobustZScoreNorm` 统计量与截断行为
- 严格只在完全匹配的 `(datetime, instrument)` 键上做对齐

### 风险 2：端到端梯度在所选特征链路上消失

缓解方式：

- 在攻击前先验证原始输入梯度
- 保持 smoke test 样本量较小，降低排查成本
- 当 clean 闸门失败时，检查特征级梯度大小

### 风险 3：原始 `volume` 的扰动由于量纲问题被放大或过弱

缓解方式：

- 为 `volume` 单独设置预算与 floor
- 将价格和成交量的实际扰动比率分别汇报

## 决策记录

以下决策在版本 1 中固定：

- 在重建成功前不重训练
- 攻击空间为原始 `OHLCV`
- 使用最大化 `MSE` 的非定向攻击目标
- 使用基于当前值的相对扰动预算
- 扰动预算分为价格组和成交量组
- 从测试集匹配样本中随机抽样
- 第一轮不接入组合收益与回测评估
