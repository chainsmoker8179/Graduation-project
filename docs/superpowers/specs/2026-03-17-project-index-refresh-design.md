# 项目实验总览索引重构设计

## 目标

将 `reports/项目实验总览与文档索引.md` 重构为一份面向项目维护者的导航型总览文档，重点服务于“快速恢复上下文、辨认当前有效基线、区分过程版本与正式版本”的需求。

## 范围

本轮只修改一份目标文档，并补充与其直接相关的导航结构，不改动其他实验报告正文。

## 设计原则

1. 以主仓库当前状态为准，不再使用旧 hidden worktree 作为默认口径。
2. 优先回答“现在该看什么”，而不是完整复述所有历史过程。
3. 把 Markdown 文档按角色分组：方法设计、梯度验证、样本级攻击、组合级攻击、证据补强、辅助文档。
4. 明确标记哪些文档是当前正式基线，哪些只是过程性版本。
5. 保留实验阶段主线，但压缩重复叙述，强调阶段衔接与当前结论。

## 预期结构

1. 文档目的与使用方式
2. 当前有效基线
3. Markdown 文档索引
4. 实验主线与阶段衔接
5. 当前项目状态判断
6. 推荐阅读路径
