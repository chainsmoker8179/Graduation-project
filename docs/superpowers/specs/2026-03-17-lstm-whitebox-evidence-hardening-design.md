# LSTM 白盒攻击单模型证据补强设计

## 1. 背景

当前仓库已经完成 legacy LSTM 在原始 `OHLCV` 输入上的白盒攻击链路，并积累了三层结果：

1. 样本级 smoke 与 expanded attack，证明 `FGSM/PGD` 可沿端到端输入链路稳定提升预测误差。
2. `5%` 每日局部攻击、多随机种子回测，证明组合级退化方向稳定。
3. `1% / 5% / 10%` 攻击比例对比，证明退化强度随攻击覆盖率上升而增强。

现阶段缺口不在于“是否存在退化”，而在于论文证据尚未完全闭合。已有结果主要依赖均值、标准差和方向一致性，仍缺少严格的显著性检验、排序破坏的机制分析，以及一套可直接进入论文主文与答辩的系统图表。

## 2. 目标

本轮工作只聚焦 legacy LSTM 单模型，把已有攻击结果升级为一条完整、可答辩、可写论文的证据链。最终需要同时回答三个问题：

1. 攻击后收益与排序退化是否具有统计显著性。
2. 组合退化是否能够由排序质量破坏来解释，而不只是若干回测指标的偶然波动。
3. 现有结果能否整理为主文聚焦、附录补充的论文呈现结构。

## 3. 范围与边界

### 3.1 本轮纳入范围

- 模型固定为 legacy LSTM。
- 主实验口径固定为：`2025-01-01` 至 `2025-10-31`、每日攻击 `5%` 股票、`seed=0..4`、攻击目标为最大化 `MSE`、攻击器为 `FGSM/PGD`。
- 主对照固定为 `partial_clean`，避免把局部 clean 替换误差与真实攻击效应混淆。
- 将 `1% / 5% / 10%` 比例扫描保留为稳健性与趋势证据，优先放入附录或补充材料。

### 3.2 本轮暂不纳入范围

- 不扩展到 Transformer、GRU、ALSTM 等多模型横向对比。
- 不新增其他攻击目标或预算扫描。
- 不重构现有回测框架，不引入新的组合构建策略。

## 4. 推荐证据结构

### 4.1 主文证据

主文只保留最稳定、最容易解释的主线结果：

1. `5%` 多随机种子回测的主结果表。
2. 日度超额收益与日度 `RankIC` 的配对显著性检验。
3. 排序机制分析，包括分数相关性、Top-K 重合率、排名位移等。
4. 几张核心图：累计超额收益曲线、多 seed 退化图、排序破坏图。

### 4.2 附录证据

附录承接“结论是否稳健”的问题：

1. `1% / 5% / 10%` 比例扫描的趋势图与表。
2. seed 级明细表。
3. 更完整的置信区间、显著性和附加诊断结果。

这样的结构可以避免主文被大量比例和 seed 细节稀释，同时保留足够的复核材料。

## 5. 方法设计

### 5.1 统计显著性层

显著性分析不直接对 `n=5` 个 seed 的汇总指标做简单检验，而是回到日度序列进行配对比较。主比较对象为：

- `partial_fgsm` vs `partial_clean`
- `partial_pgd` vs `partial_clean`

第一优先级指标：

- 日度超额收益 `excess_return_with_cost`
- 日度 `RankIC`

推荐统计方法：

- 主检验：Wilcoxon signed-rank（配对、非参数）
- 区间估计：bootstrap 置信区间
- 汇总指标如年化超额收益、信息比率、最大回撤：采用基于日度序列的 block bootstrap 估计差值区间，而不是直接把单个汇总值拿去做显著性检验

预期输出：

- `p-value`
- 效应量或中位差/均值差
- `95%` 置信区间
- 方向一致性统计

### 5.2 排序机制层

仅展示收益退化还不足以解释机制，因此需要补一层“排序被如何破坏”的分析。推荐诊断包括：

- 攻击前后分数的日度 Spearman 相关
- 攻击前后 Top-K 重合率
- 排名位移分布
- 被攻击股票进入/离开组合候选集的比例

其作用是把“分数发生扰动”与“组合构建退化”连接起来，说明攻击并非只改变了少量分值，而是系统性破坏了横截面排序结构。

### 5.3 图表与表格层

图表沿用现有 `scripts/plotting` 体系，补充主文需要的论文级图形。推荐产物如下：

- 主图 1：`clean / FGSM / PGD` 的累计超额收益曲线
- 主图 2：`5%` 多 seed 下四个核心指标的退化分布图
- 主图 3：日度 `RankIC` 或分数相关性的排序破坏图
- 主图 4：Top-K 重合率或排名位移图

主表建议包含：

- 主结果表：均值、标准差、方向一致性
- 显著性检验表：收益与 `RankIC` 的检验结果
- 排序机制表：Spearman、Top-K overlap、rank shift
- 稳健性表：`1% / 5% / 10%` 趋势结果

## 6. 数据与产物约束

本轮工作应优先复用现有实验产物，不重新设计数据格式。关键输入包括：

- `reports/partial_attack_backtest_multiseed_ratio5_union/seed_*/daily_comparison.csv`
- `reports/partial_attack_backtest_multiseed_ratio5_union/multiseed_seed_metrics.csv`
- `reports/partial_attack_backtest_multiseed_ratio5_union/multiseed_summary_stats.csv`
- `reports/partial_attack_ratio_sweep_multiseed/ratio_sweep_seed_metrics.csv`
- `reports/partial_attack_ratio_sweep_multiseed/ratio_sweep_summary_stats.csv`

新的分析脚本应输出结构化中间结果，便于图表和中文报告复用，而不是只生成一份不可复算的 Markdown 文本。

## 7. 交付物

本轮设计对应的最终交付物包括：

1. 一个显著性分析结果目录，包含检验表、区间估计和可复用的中间 CSV/JSON。
2. 一个排序机制分析结果目录，包含相关性、Top-K overlap、rank shift 等统计结果。
3. 若干张纳入现有 `paper/slide` 双尺寸体系的图。
4. 一份中文实验报告，总结主结果、显著性结论、排序机制和论文放置建议。

## 8. 风险与控制

### 8.1 风险

- 不同指标的统计单位不一致，若直接对年化收益或最大回撤做 seed 级显著性检验，容易得到不可靠结论。
- `partial_clean - reference_clean` 的残余偏差若处理不当，可能污染攻击解释。
- 排序机制分析若只做全局平均，可能掩盖不同交易日的横截面差异。

### 8.2 控制策略

- 所有显著性结论优先基于日度配对序列给出。
- 主对照统一使用 `partial_clean`。
- 中间结果按日和按 seed 都保留，避免只剩汇总值。
- 图表和表格共享同一份中间分析产物，避免口径漂移。

## 9. 成功标准

若本轮完成后满足以下四点，即认为 legacy LSTM 的单模型论文证据已经基本闭合：

1. 日度超额收益和日度 `RankIC` 的退化具有清晰的显著性证据。
2. 年化收益、信息比率和最大回撤具有合理的区间估计。
3. 排序机制指标能解释组合层退化，而不是只停留在现象描述。
4. 形成一套主文聚焦 `5%` 多 seed、附录补充比例趋势的中文论文材料。
